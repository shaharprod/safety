הגדרת Google OAuth — SafetyOS

הפעלת כניסה עם Google בדף שער האתר (GateControl)

☁️ שלב א׳ — יצירת Client ID ב-Google Cloud

1
פתח את console.cloud.google.com וודא שאתה בפרויקט הנכון (או צור פרויקט חדש).
2
בתפריט הצדדי: APIs & Services → Credentials
3
לחץ Create Credentials → OAuth 2.0 Client ID
4
Application type: Web application
Name: SafetyOS (או כל שם אחר)
5
תחת Authorized JavaScript origins הוסף:
https://your-app.onrender.com # כתובת Render http://localhost:5173 # פיתוח מקומי
⚠️ החלף your-app.onrender.com בכתובת האמיתית של השירות שלך ב-Render.
6
לחץ Create ← העתק את ה-Client ID שמופיע (נראה כך):
123456789012-abcdefghijklmnopqrstuvwxyz.apps.googleusercontent.com

🖥️ שלב ב׳ — הגדרה מקומית (פיתוח)

1
בתיקיית frontend/ צור קובץ בשם .env (אם לא קיים):
# frontend/.env VITE_GOOGLE_CLIENT_ID=YOUR_CLIENT_ID_HERE
2
הפעל מחדש את שרת הפיתוח:
cd frontend npm run dev
⚠️ Vite טוען משתני סביבה רק בהפעלה — שינוי ב-.env מחייב הפעלה מחדש של npm run dev.
3
ודא שהקובץ .env מופיע ב-.gitignore ולא מועלה ל-GitHub:
# .gitignore frontend/.env frontend/.env.local

🚀 שלב ג׳ — הגדרה ב-Render (production)

1
כנס ל-dashboard.render.com ובחר את השירות של SafetyOS.
2
עבור ל-Environment → Environment Variables
3
לחץ Add Environment Variable והוסף:
VITE_GOOGLE_CLIENT_ID = your-client-id.apps.googleusercontent.com
4
לחץ Save Changes ← Render יפעיל deploy אוטומטי עם הערך החדש.
ℹ️ Vite מוצא את VITE_* בזמן ה-build. לכן כל שינוי במשתנה זה מחייב build חדש — Render עושה זאת אוטומטית.

✅ אימות — כך בודקים שהכל עובד

1
כנס לאפליקציה ← עבור לדף שער הכניסה (🔑 כניסה)
2
בחר בלשונית "כניסה עם Google" — כפתור Sign In With Google אמור להופיע.
אם עדיין מופיעה הודעת השגיאה — ודא ש-VITE_GOOGLE_CLIENT_ID הוגדר והאפליקציה נבנתה מחדש.
3
התחבר עם חשבון Google שמוגדר לעובד ב-ניהול עובדים עם שדה אימייל Google.
מה קורה מאחורי הקלעים:
הדפדפן שולח את ה-credential token של Google ל-POST /api/gate/check-google. השרת מפענח אותו, מחפש עובד עם אותו google_email, ומחזיר את סטטוס הגישה לאתר.
